Guía del Sistema NeuroTEA

Documento explicativo para entender cómo trabajo y cómo usar las herramientas de verificación

1. ¿Qué hicimos juntos?

Durante nuestra sesión de trabajo, realizamos las siguientes mejoras al sistema:

Correcciones realizadas:
  • Eliminamos 3 funciones que estaban duplicadas (esto podía causar comportamientos inesperados)
  • Actualizamos la documentación técnica para que refleje el estado real del sistema
  • Creamos un sistema de pruebas automatizadas con 76 verificaciones
  • Creamos 3 "agentes especializados" que te ayudarán a verificar el sistema

2. ¿Qué son las pruebas automatizadas?

Imagina que tienes una lista de 76 cosas que verificar cada vez que alguien toca el sistema. Sería muy tedioso hacerlo manualmente, ¿verdad?

Las pruebas automatizadas son exactamente eso: una lista de verificaciones que la computadora ejecuta automáticamente. En segundos, el sistema revisa:

  • Que los cálculos de dinero sean correctos
  • Que los créditos de paquetes funcionen bien
  • Que al eliminar algo, se elimine de todas partes
  • Que las sesiones grupales dividan el dinero correctamente
  • Y 72 cosas más...
Resultado esperado:
Cuando todo funciona bien, verás un mensaje que dice:
✅ Pasadas: 76 | ❌ Fallidas: 0

Si alguna prueba falla, sabremos exactamente qué función hay que revisar.

3. ¿Qué son los agentes especializados?

Los agentes son como "asistentes expertos" que yo (Claude) puedo usar para tareas específicas. Cada uno sabe exactamente qué revisar.

Agente ¿Qué hace? ¿Cuándo usarlo?
/test-neurotea Ejecuta las 76 pruebas automáticas Después de cualquier cambio en el código
/verify-neurotea Verifica que todos los archivos estén completos y no falte nada Después de cambios grandes o cuando algo parece raro
/audit-coherence Audita que el sistema siga las "reglas de oro" (coherencia) Antes de hacer un backup importante

4. ¿Cómo usar los agentes?

Es muy sencillo. Cuando estés conversando conmigo (Claude), simplemente escribe el comando:

Ejemplo:

Tú: /test-neurotea

Yo ejecutaré las pruebas y te diré si todo está bien o si hay algún problema.

5. ¿Qué son las "reglas de oro" del sistema?

El sistema NeuroTEA sigue 3 principios fundamentales que aseguran que todo funcione correctamente:

Regla 1: Reversibilidad

"Todo se puede deshacer"

Si registras una sesión por error, puedes eliminarla. Si confirmas un pago incorrectamente, puedes revertirlo. El sistema siempre permite volver atrás.

Regla 2: Coherencia

"El mismo dato se ve igual en todas partes"

Si una sesión aparece en el Registro Diario, también aparece en el Resumen, en la Rendición de Cuentas, y en las Transferencias. No puede estar en un lugar y no en otro.

Regla 3: Cálculo dinámico

"Los saldos se calculan, no se guardan"

El saldo de caja no es un número guardado que se va sumando. Cada vez que lo necesitas, el sistema lo calcula desde cero: inicial + ingresos - egresos - pagos. Esto evita errores de acumulación.

6. ¿Qué pasa si algo falla?

Si una prueba falla:
  1. El sistema te mostrará exactamente cuál prueba falló
  2. Te indicará qué función está afectada
  3. Puedes pedirme que investigue y corrija el problema

7. ¿Cuándo debo usar cada herramienta?

Situación Qué hacer
Pediste un cambio pequeño en el código Usa /test-neurotea
Agregaste una nueva funcionalidad Usa /test-neurotea y luego /verify-neurotea
Algo no funciona como esperabas Usa /verify-neurotea para diagnóstico
Vas a hacer un backup importante Usa /audit-coherence primero
Importaste datos de otro día Usa /test-neurotea para verificar integridad

8. Resumen ejecutivo

Lo que tienes ahora:
  • 76 pruebas automáticas que verifican que el sistema funcione correctamente
  • 3 agentes especializados que puedes invocar con comandos simples
  • Documentación actualizada que refleja el estado real del sistema
  • Sistema limpio sin funciones duplicadas

Beneficio principal: Ahora puedes hacer cambios al sistema con confianza. Después de cada modificación, las pruebas te dirán si algo se rompió, antes de que lo descubras en producción.
